Areas de git

Working directory

El area de trabajo corresponde a la carpeta donde iniciamos el repositorio y también donde vamos a estar realizando los cambios de nuestro proyecto. Practicamente no necesitamos utilizar ningún comando, salvo en muy pocas situaciones.

Staging Areab(INDEX)

El area de control de cambios se utiliza para agregar los archivos a los que queremos realizarles seguimiento, de esta manera GIT nos puede notificar cuando estos fueros modificados y el estado de los mismos. Para lograr esto, se realiza un Snapshot (captura de pantalla) del codigo del archivo. Los archivos que no sean agregados al INDEX, no podran ser restablecidos y tampoco se llevara control de los mismos.

Repository

Corresponde al lugar donde se almacenan las capturas en formato BLOB (Binary Large Object File generando un numero HASH utilizando la informacion de nombre de usuario, correo, titulo del mensaje, registro de cambios, ademas de la fecha y hora en la que se realiza la confirmacion. Esto es necesario para registrar la relacion que existe entre dicho commit y las confirmaciones de cambios anteriores
